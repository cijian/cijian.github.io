与远程仓库版本冲突

最简单最傻逼的方法就是删除当前项目再clone一次下来。但由于网络问题项目clone比较慢。

1、git stash
git stash可用来暂存当前正在进行的工作，比如想pull 最新代码，又不想加新commit，或者另外一种情况，为了fix一个紧急的bug,先stash, 使返回到自己上一个commit, 改完bug之后再stash pop, 继续原来的工作。

2、git reset
reset是指将当前head的内容重置，不会留任何痕迹。可能你没提交的修改也被重置

重置上一次的提交

最新的3次提交全部重置，就像没有提交过一样。

3、revert
revert是撤销某次提交，但是这次撤销也会作为一次提交进行保存(这样就不会丢失原来修改过，但是没有提交的内容？)。
